# Execution Harnesses

Harnesses are small task-local checklists that turn L0 cognition into execution discipline.

They are not policy runtimes. A harness should fit in one model context refresh and answer:

- What L0 state must be loaded?
- What skills are relevant?
- What invariants must be preserved?
- What patterns are forbidden?
- What review is required before completion?

Use one primary harness per task and add one secondary harness only when the task crosses boundaries.
